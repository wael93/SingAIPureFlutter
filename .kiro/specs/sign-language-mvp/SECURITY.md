# إرشادات الأمان - نظام ترجمة الإشارة

## المبادئ التوجيهية للأمان

### ما يجب عدم رفعه أبداً إلى GitHub:

1. **المفاتيح السرية (Secrets)**
   - API Keys
   - Access Tokens
   - Client Secrets
   - Encryption Keys

2. **المعلومات الحساسة**
   - كلمات المرور
   - بيانات الاعتماد
   - شهادات SSL/TLS

3. **ملفات التكوين الحساسة**
   - `.env` (حتى لو كانت فارغة)
   - ملفات الإعداد التي تحتوي على بيانات حقيقية

4. **النماذج المدربة**
   - ملفات النماذج الكبيرة (`.tflite`, `.bin`)
   - النماذج التي تحتوي على بيانات حساسة

### الملفات الآمنة للرفع:

1. **كود التطبيق**
   - جميع ملفات `.dart` في `lib/`
   - ملفات الاختبار

2. **الملفات التكوينية العامة**
   - `pubspec.yaml`
   - `analysis_options.yaml`
   - `app_config.dart` (بدون بيانات حقيقية)

3. **الملفات الوثائقية**
   - `README.md`
   - `SECURITY.md`
   - ملفات التصميم والمتطلبات

### إجراءات الأمان:

1. **قبل كل دفعة (commit)**
   - تأكد من عدم وجود ملفات `.env`
   - تأكد من عدم وجود مفاتيح في الكود
   - استخدم `git status` للتحقق

2. **استخدام `.gitignore`**
   - تأكد من أن `.gitignore` يحتوي على جميع الملفات الحساسة
   - تحقق من أن الملفات المهمة ليست مهملة

3. **التحقق من المحتوى**
   ```bash
   # تحقق من الملفات التي سيتم رفعها
   git status
   
   # تحقق من المحتوى قبل الرفع
   git diff --cached
   ```

### نموذج `.env` آمن:

```bash
# هذا الملف مهمل في .gitignore
# استخدم هذا كنموذج فقط

# إعدادات التطوير (بدون بيانات حقيقية)
API_KEY=your_api_key_here
DATABASE_URL=your_database_url_here
```

### في حال رفع بيانات حساسة عن طريق الخطأ:

1. **لا ترفع البيانات** - أوقف العملية فوراً
2. **احذف الملف** من الكوميت الحالي
3. **استخدم `git reset`** لإلغاء الكوميت
4. **غيّر البيانات** فوراً إذا كانت حساسة

### أدوات المساعدة:

```bash
# تحقق من الملفات الكبيرة
git ls-files | xargs ls -lh

# تحقق من الملفات المخفية
git ls-files --others --ignored --exclude-standard

# تحقق من المحتوى قبل الرفع
git diff --cached --name-only
```

## ملاحظات مهمة:

- هذا المشروع يستخدم **Local-First Architecture** - لا توجد بيانات تُرسل للخوادم
- جميع النماذج تُخزن محلياً على الجهاز
- لا توجد مفاتيح API مطلوبة للنسخة الأولية (MVP)
- التشفير يتم باستخدام مفاتيح الجهاز الأصلية (Android Keystore / iOS Keychain)
